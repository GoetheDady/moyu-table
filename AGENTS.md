# 仓库协作规则

- 回复用到专业术语的时候，需要具体解释一下。
- 写代码时需要补充必要注释。方法需要完整的方法注释，说明用途、参数、返回值；如果存在副作用、边界条件或关键约束，也需要一并说明。普通行内注释应解释意图、约束或不容易一眼看懂的逻辑，避免重复描述代码表面行为。

## Agent skills

### Issue tracker

Issue 和 PRD 使用 GitHub Issues 跟踪，仓库地址是 `git@github.com:GoetheDady/moyu-table.git`。详见 `docs/agents/issue-tracker.md`。

### Triage labels

本仓库使用默认的五个分诊标签，并在文档中维护中文说明。详见 `docs/agents/triage-labels.md`。

### Domain docs

本仓库使用 single-context（单上下文）领域文档布局。详见 `docs/agents/domain.md`。
